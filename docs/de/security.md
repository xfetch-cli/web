# Sicherheitsrichtlinie

## Melden von Sicherheitslucken

Wenn Sie eine Sicherheitslucke in **xfetch** entdecken, melden Sie diese bitte verantwortungsvoll per E-Mail:

**E-Mail:** `x@xscriptor.com`

### Was Sie angeben sollten

Bitte geben Sie bei der Meldung eines Sicherheitsproblems Folgendes an:

1. **Beschreibung** — Eine klare Erklarung der Schwachstelle
2. **Typ** — Um welche Art von Sicherheitsproblem handelt es sich? (z.B. Pufferuberlauf, Injektion, Privilegieneskalation)
3. **Schritte zur Reproduktion** — Detaillierte Schritte zum Auslosen der Schwachstelle
4. **Auswirkungen** — Wie schwerwiegend ist das Problem? Was konnte ein Angreifer tun?
5. **Betroffene Versionen** — Welche Versionen von xfetch sind betroffen?
6. **Losungsvorschlag** (optional) — Wenn Sie einen Vorschlag zur Behebung haben

### Richtlinien

- **Offnen Sie keine** offentlichen GitHub-Issues fur Sicherheitslucken
- **Offenbaren Sie die** Schwachstelle nicht offentlich, bis ein Fix veroffentlicht wurde
- **Geben Sie den** Entwicklern angemessene Zeit, das Problem vor der offentlichen Offenlegung zu beheben
- In der Regel streben wir eine Ruckmeldung innerhalb von **7 Tagen** und einen Fix innerhalb von **30 Tagen** fur kritische Probleme an

## Sicherheitsempfehlungen fur Benutzer

Obwohl xfetch sicherheitsbewusst entwickelt wurde, beachten Sie bitte folgende Empfehlungen:

1. **Aktualisiert bleiben** — Verwenden Sie immer die neueste Version von xfetch
2. **Downloads uberprufen** — Prufen Sie Prüfsummen beim Herunterladen von Binarprogrammen
3. **Konfigurationsdatenschutz** — Seien Sie vorsichtig beim Teilen benutzerdefinierter Konfigurationen, die sensible Pfade enthalten konnen
4. **Netzwerkfunktionen** — Bei Verwendung von IP-Erkennungsfunktionen sollten Sie sich der moglichen Auswirkungen auf die Privatsphare bewusst sein
5. **Plugins von Drittanbietern** — Verwenden Sie nur Plugins aus vertrauenswurdigen Quellen
